## Source Categories

## Source Categories

Preferred source quality, in order:
1. **Academic papers** (peer-reviewed, arXiv) — strongest provenance
2. **Post-mortem incidents** (Rekt News, SlowMist, protocol post-mortems) — grounded in real losses
3. **Research blogs** (Flashbots, Paradigm, a16z crypto, Chainalysis, Nansen Query) — industry-vetted
4. **Public dashboards** (Dune, Flipside) — reproducible, data-backed
5. **Open-source detection code** (MEV-Inspect, Forta bots, slither detectors) — runnable reference
6. **Product documentation** (GoPlus API docs, De.Fi Scanner spec) — useful but vendor-biased

## Rules

- Heuristics with no citation → label `unverified-heuristic` in code + TODO to find prior art, or remove
- Multiple sources converging on the same technique → cite the strongest 2-3, not all
- Thresholds without a derivation story → rejected in review
